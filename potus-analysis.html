About the project

This semester, I used a few text analysis methods on the POTUS Speech Corpus for a final project in DS 5559 Exploratory Text Analytics. Everything was done in Jupyter Notebook; we created Topic Models with gensim and wrote Sentiment Analyses using vaderSentiment.

The motivation behind this project was to discover some of the trending national concerns during each president's term. Ideally, using this analysis we should be able to map out a history of our nation with respect to the globe.

You can find view code for the project on github here.

Topic Modeling

The process for bringing the corpus from F0 to F4 is quite lengthy. The corpus comes in a zipped directory of subdirectories, one for each president. Each subdirectory contains a a few text files, one for each speech that president made during his term. The format of each speech was in XML format, which made it quite easy to parse. Once all of this data was propery parsed into a pandas DataFrame, we cleaned up the text for any bad characters, and tokenized so we could create bigram and trigram models for our data.

Then we remove any stopwords, lemmatize, and form a dictionary of our vocabulary. Once this is done, we feed our F4 data into the gensim topic model wrapper. Sentiment Analysis

The process for feeding our data into VADER's Sentiment Analysis model is much simpler. Since the model operates best on the sentence level, we simply import all of our corpus data into an appropriate DataFrame, and we concatenate every president's speeches together, and split each combined speech into a list of sentences.
